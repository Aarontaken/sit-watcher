# SitWatcher — 久坐提醒桌面应用设计文档

## 概述

SitWatcher 是一款 macOS 菜单栏应用，用于提醒用户定时起身活动。应用平时安静地驻留在菜单栏，通过渐进式提醒机制确保用户不会忽略提醒——从温和的系统通知逐步升级到全屏遮罩。

## 目标平台

- macOS（Menu Bar Only App，不显示在 Dock 中）
- 技术栈：Swift + SwiftUI
- 最低支持版本：macOS 14 (Sonoma)

## 核心架构

应用由四个核心模块组成：

### 1. TimerEngine（计时引擎）

负责倒计时逻辑和用户活动检测。

- 默认 30 分钟循环倒计时
- 倒计时结束后触发提醒流程
- 用户确认休息后（或 L3 被关闭后）自动重新开始计时

**Idle 检测**：利用 `CGEventSource` 获取用户空闲时间，结合鼠标位置变化判断真实活动。

- 记录上次鼠标位置，每秒对比当前位置
- 移动距离 < 10px → 视为脚本/keep-alive 行为，不重置 idle 计时
- 移动距离 >= 10px 或有键盘输入 → 视为真人操作，重置 idle 计时
- 用户 idle 超过 5 分钟 → 暂停计时器并重置，视为用户已离开
- 用户回来后（检测到真实输入）→ 自动开始新一轮计时

### 2. ReminderEscalator（渐进提醒器）

三级渐进式提醒，倒计时结束后依次升级：

**Level 1 — 轻声提醒（立即触发）**

- 菜单栏图标从绿色 ● 变为黄色 ● 呼吸闪烁
- 发送一条 macOS 系统通知：「已经坐了 30 分钟了，起来活动一下吧」
- 播放轻提示音（可关闭）
- 不打断任何工作流
- 用户响应方式：点击系统通知，或打开菜单栏面板点击「重置」→ 关闭提醒并重置计时器

**Level 2 — 浮窗提醒（L1 后 2 分钟未响应）**

- 屏幕右上角弹出浮窗，置顶显示（NSWindow level 设为 floating）
- 菜单栏图标变为橙色 ●
- 浮窗内容：emoji + 「该站起来了！」+ 已坐时长
- 两个按钮：
  - 「好的，我去活动」→ 关闭所有提醒，重置计时器
  - 「稍后 5 分钟」→ 关闭浮窗，5 分钟后从 L1 重新开始升级（每周期限 1 次）
- 浮窗不遮挡主要工作区域

**Level 3 — 全屏遮罩（L2 后 2 分钟未响应）**

- 半透明遮罩覆盖整个屏幕（NSWindow level 设为 screenSaver）
- 背景高斯模糊
- 菜单栏图标变为红色 ●
- 中央显示：emoji + 「起来走走吧」+ 已坐时长
- 唯一按钮：「我已经站起来了」→ 关闭遮罩，重置计时器
- 这是最终手段，确保用户无法继续忽略

### 3. MenuBarPanel（菜单栏面板）

点击菜单栏图标展开的控制面板，深色主题，从上到下：

1. **标题栏**：应用名「SitWatcher」+ 当前状态指示（绿点 + "专注中"）
2. **环形进度条**：渐变色圆环（绿→青），中央大字显示剩余时间（如 18:42）
3. **快捷控制**：三个按钮
  - 暂停/继续：暂停当前计时
  - 跳过：跳过本轮，立即开始下一轮计时
  - 重置：重置当前倒计时
4. **今日统计**：
  - 已休息次数（用户主动确认的次数）
  - 被打断次数（升级到 L3 的次数）
  - 累计专注时长
5. **底部**：设置入口 + 退出按钮

**菜单栏图标状态颜色**：

- 绿色：专注中（正常倒计时）
- 黄色闪烁：L1 提醒中
- 橙色：L2 浮窗阶段
- 红色：L3 全屏阶段
- 灰色：已暂停

### 4. SettingsManager（设置管理）

通过面板底部「设置」入口打开设置视图。


| 设置项     | 默认值   | 范围/说明                    |
| ------- | ----- | ------------------------ |
| 提醒间隔    | 30 分钟 | 15-120 分钟                |
| L2 升级延迟 | 2 分钟  | L1 被忽略后多久弹出浮窗            |
| L3 升级延迟 | 2 分钟  | L2 被忽略后多久全屏遮罩            |
| Idle 阈值 | 5 分钟  | 无真实操作多久算"离开了"            |
| 鼠标移动阈值  | 10 px | 低于此距离的移动视为脚本行为           |
| 提示音     | 开启    | L1 通知时播放轻提示音             |
| 开机自启动   | 关闭    | 登录时自动启动（通过 SMAppService） |


持久化：所有设置存储在 `UserDefaults` 中。

## 视觉设计

- **主题**：深色（Dark），与 macOS 系统深色模式一致
- **配色**：
  - 主色：绿→青渐变（#4ade80 → #22d3ee）
  - 背景：深蓝灰（#0f172a, #1e293b）
  - 文字：亮灰（#f1f5f9）、中灰（#94a3b8）
  - 状态色：绿（正常）、黄（L1）、橙（L2）、红（L3）、灰（暂停）
- **图标**：使用 SF Symbols，无需额外图片资源
- **动画**：菜单栏图标呼吸闪烁、环形进度条平滑变化、浮窗淡入淡出

## 数据管理

- 设置数据：`UserDefaults`
- 今日统计：内存中维护，每日零点自动重置
- 不做历史记录持久化，保持应用简洁

## 应用生命周期

```
启动 → 开始计时 → 倒计时结束 → L1 提醒
                                    ↓
                              用户响应？──是──→ 重置计时，回到「开始计时」
                                    ↓ 否（2min）
                                L2 浮窗
                                    ↓
                      用户点「好的」？──是──→ 重置计时
                      用户点「稍后」？──是──→ 5 分钟后从 L1 重新开始
                                    ↓ 否（2min）
                                L3 全屏遮罩
                                    ↓
                        用户点「我已站起来」→ 重置计时
```

Idle 检测贯穿整个生命周期：用户离开（idle > 5min）→ 暂停并重置 → 用户回来 → 开始新一轮计时。

## 「稍后提醒」机制

- 仅 L2 浮窗提供「稍后 5 分钟」按钮
- 点击后推迟一次，5 分钟后从 L1 重新开始升级流程
- 每个周期最多推迟 1 次，防止无限拖延

## 资源占用

- 应用体积：约 5-10MB
- 常驻内存：约 15-20MB
- 无网络请求，无外部依赖
- CPU 占用可忽略（仅计时器 tick 和偶尔的鼠标位置检测）

## 权限需求

- **通知权限**：发送系统通知（UNUserNotificationCenter）
- **辅助功能权限**：不需要（CGEventSource 不需要辅助功能权限即可获取 idle 时间）

## 不做的事情

- 不做跨平台支持
- 不做历史数据统计/图表
- 不做拉伸动作引导
- 不做全局快捷键（避免冲突）
- 不做网络同步
